# WebUI - User Interface Design (Search-First)

## Design Philosophy

**Search-First Approach**: The main page is a search interface where users can:
1. Search their tracked products by name
2. Paste a product URL to check if it's tracked
3. Add new products when search returns no results

This creates a simpler mental model: "Search for what you want, track it if you don't have it yet."

### Backend Logic (`/search` endpoint)

```python
@login_required
def search(request):
    query = request.POST.get('query', '').strip()

    # Detect if query is a URL
    if query.startswith('http://') or query.startswith('https://'):
        return handle_url_search(request, query)
    else:
        return handle_name_search(request, query)


def handle_url_search(request, url):
    """Handle URL-based search."""
    # Check if product exists for this user
    product = Product.objects.filter(url=url, user=request.user).first()

    if product:
        # Already tracking - show product detail
        return render(request, 'search/url_tracked.html', {'product': product})
    else:
        # Not tracking - show "Add to tracking?" prompt
        domain = urlparse(url).netloc

        # Check if store is supported
        supported_stores = ['amazon.com', 'ebay.com', 'walmart.com', 'bestbuy.com', 'target.com']
        if not any(store in domain for store in supported_stores):
            return render(request, 'search/url_unsupported.html', {'url': url, 'domain': domain})

        # Show add prompt
        return render(request, 'search/url_not_tracked.html', {'url': url, 'domain': domain})


def handle_name_search(request, query):
    """Handle name-based search."""
    products = Product.objects.filter(
        user=request.user,
        active=True,
        name__icontains=query
    ).order_by('-last_viewed')[:20]

    if products.exists():
        return render(request, 'search/results.html', {'products': products, 'query': query})
    else:
        # 0 results - show "add URL" prompt
        return render(request, 'search/no_results.html', {'query': query})
```

## Design Improvements Over v1

**1. Search-First is More Intuitive**
- Primary action (search) is obvious
- Adding products is contextual, not forced upfront
- Matches user mental model: "Let me check if I have this"

**2. URL Detection is Smart**
- Automatically detects URL vs. name search
- Different handling for each type
- Clear feedback for each scenario

**3. Progressive Disclosure**
- Start simple (just search)
- Show complexity only when needed (add form)
- Loading states explain what's happening

**4. Reduced Cognitive Load**
- One input field (not separate "add" flow)
- Search AND add use same starting point
- Less navigation, more focus

**5. Better Error Handling**
- Clear messages for unsupported stores
- Helpful suggestions for 0 results
- Graceful degradation for failures
